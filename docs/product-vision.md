# P&ID-Agent 产品主旨

## 一句话定位

P&ID-Agent 是一款比 AutoCAD 更轻、更专注的 P&ID 工艺流程图软件：工程人员可以自由画图和修改，Agent 可以读取同一份语义化图纸，使用同一套单位图例协助创建、修改、解释和检查流程图。

它不是通用 CAD 的缩小版，而是围绕工艺流程图重新设计的专用工具。

## 要解决的问题

传统 CAD 功能广泛，但在工艺流程图场景中存在以下负担：

- 软件体量和学习成本较高；
- 工艺符号、管线、位号和设备关系通常只是几何图形，缺少机器可理解的语义；
- 单位内部图例难以沉淀为可复用、可校验的标准库；
- 工程人员的人工修改和 Agent 的自动生成容易形成两套数据；
- 从工艺说明到整套流程图仍需要大量重复布置、连线和标注工作。

P&ID-Agent 的核心价值是把“人工 P&ID 编辑器”和“Agent P&ID 工具”合并为同一个工作区。

## 目标用户

- 工艺设计人员；
- 设备、仪表、管道相关工程人员；
- 需要维护单位图例和设计标准的工程团队；
- 使用云端或本地大模型辅助设计的用户。

## 三种核心工作方式

### 1. 人工自由绘图

用户从单位符号库拖放设备、阀门、仪表和接口，绘制正交管线，填写位号、介质、管径和说明，并完成移动、复制、删除、对齐、分组和图层管理。

### 2. Agent 自动生成

用户上传或输入工艺设计理念、流程描述、设备清单和内部规范。Agent 读取单位图例、图纸规则和当前文档，生成结构化绘图事务，自动完成设备布置、管线连接、位号和标注。

### 3. 人机协同修改

用户可以先让 Agent 生成草图，再手工调整；也可以先人工画一部分，再要求 Agent 补充、重排、检查或解释。人工与 Agent 始终操作同一份文档，不维护两套状态。

## 符号库是共同语言

符号库和图例库必须同时服务于人和 Agent。

每个符号不只是一个 SVG 图形，还应包含：

- 稳定的符号 key；
- 中文名称、分类和工程含义；
- 默认尺寸和显示形状；
- 可连接端口、端口方向和端口类型；
- 允许的旋转和缩放规则；
- 位号规则及可填写属性；
- Agent 可读取的用途说明和连接约束。

单位后续提供参考图例时，应优先替换声明式符号定义，而不是修改编辑器内核。

## 目标成图能力

最终应能制作完整、复杂且可继续编辑的 P&ID，包括：

- 多工艺系统和多区域流程；
- 储罐、容器、泵、风机、换热设备、柜体等设备；
- 阀门、仪表、执行机构和系统接口；
- 大量正交管线、分支、汇合、跨线和流向箭头；
- 按介质、系统或专业区分的颜色和线型；
- 设备位号、仪表位号、管线编号和文字标注；
- 可追踪的设备端口与管线连接关系；
- 可供 Agent 查询的场景摘要、设备表、管线表和仪表索引；
- SVG、PNG、JSON，以及后续的 PDF、DXF 导出。

目标不是生成一张不可编辑的图片，而是生成与实际复杂流程图相当的结构化工程图纸。

## 产品边界

### 必须做好

- P&ID 符号放置与管理；
- 快速、稳定的正交布管；
- 连接口吸附和连接关系维护；
- 位号、标签和工程属性编辑；
- 大图缩放、平移、选择和局部编辑；
- 单位图例导入、覆盖和版本管理；
- Agent 创建、修改、解释和规则检查；
- 人工修改后 Agent 能读取最新状态；
- 图纸导入导出和项目持久化。

### 明确不做

除非未来 P&ID 场景确实需要，否则不引入：

- 三维建模；
- 机械零件和实体建模；
- 建筑、BIM、GIS 等专业系统；
- 复杂几何约束和参数化建模；
- 渲染、材质和动画；
- 为兼容通用 CAD 而堆积的大量低频命令。

## 交互原则

- 新用户不需要先学习 AutoCAD 命令体系；
- 高频操作应通过拖放、快捷键和上下文属性完成；
- 管线优先正交，连接优先吸附到明确端口或连接节点；
- 符号、管线、分支和标签应保持语义，不退化为无意义线段；
- Agent 的每次修改都应可预览、可撤销、可追踪；
- 大模型不可绕过文档事务直接写数据库或任意执行代码；
- 软件应可在普通办公电脑和内网环境中运行。

## 技术方向

- React + TypeScript 提供轻量浏览器编辑器；
- SVG 作为主要交互渲染层，便于选择和导出；
- Python/FastAPI 提供文档服务和 Agent API；
- SQLite 作为首阶段本地持久化；
- 文档、REST、MCP、Python Client 和网页共用同一事务模型；
- OpenAI-compatible 模型接口兼容云端模型、Ollama、LM Studio 等本地服务；
- 单位规范、工艺说明和历史图纸作为后续项目知识库输入。

## 近期优先级

1. 完成真正可用的 P&ID 编辑闭环：符号、端口、正交管线、吸附、标签、撤销和大图导航；
2. 建立单位图例导入流程，将参考图例转换为声明式符号；
3. 让 Agent 能根据自然语言生成和修改带连接语义的流程；
4. 增加自动布局、避让、跨线和管线整理；
5. 增加设备表、管线表、仪表索引与图纸双向同步；
6. 增加工艺规则检查、单位标准校验和项目知识库；
7. 完善 PDF、DXF 和打印图幅能力。

## 验收标准

当用户提供一份工艺描述和单位图例后，系统应能够：

1. 生成一张结构化 P&ID 草图；
2. 正确使用指定单位符号；
3. 将设备端口、连接节点和管线连接为可查询关系；
4. 允许用户在浏览器中继续编辑；
5. 让 Agent 理解用户修改后的最新图纸；
6. 对指定设备、管线或区域执行局部修改，而不是整图重画；
7. 导出可交付、可再次导入并继续编辑的图纸。

# Issue #1 切片：折点整理与 Agent 局部修改预览

本切片将管线路径编辑和网页 Agent 从“直接提交”升级为可检查、可确认的工程工作流。

## Connector 路径编辑

选中 connector 后，属性面板新增四个操作：

- 新增偏移段：在当前最长正交段中插入一个 U 形偏移段；
- 删除偏移段：删除一个可识别的 U 形偏移段；
- 清理共线折点：移除重复点和无意义的共线中间点；
- 按端点自动整理：根据当前首尾端点重新生成紧凑正交路径。

这些操作：

- 不改变 connector id；
- 不改变 source/target 的 element_id 和 port_id；
- 将路径设置为 `manual`，确保后端保留整理后的折线；
- 仍经过服务端正交校验、图层锁定和 revision 检查。

当前自动整理只处理单条 connector，不进行设备避障、管线间距优化或整图自动布局。复杂路径仍可使用内部线段拖动继续调整。

## Agent 两阶段工作流

网页 Agent 的默认流程改为：

1. 生成事务预览；
2. 服务端在文档副本上完整执行校验；
3. 网页显示 explanation、transaction label、operation 数量和逐项摘要；
4. 用户确认应用或放弃；
5. 确认应用时仍使用预览生成时的 expected revision。

预览阶段不会写入 SQLite，不会增加 revision，也不会生成历史记录。

确认应用使用：

```http
POST /api/v2/documents/{document_id}/agent/apply
```

历史来源记录为 `llm`。

通用事务校验接口：

```http
POST /api/v2/documents/{document_id}/transactions/validate
```

返回当前 revision、下一 revision、operation 数量、结果元素数量和受影响元素 id。错误会尽量定位到：

```text
operations[3] (update_element): ...
```

## 局部修改范围

网页存在选中元素时，会将这些元素的完整结构化 JSON 附加到模型上下文，并要求：

- 优先修改选中元素和直接相连管线；
- 保留无关元素；
- 只有用户指令明确要求时才扩大修改范围。

未选择元素时，Agent 仍以整张图为范围。

## 并发保护

如果预览基于 r12，而用户或 MCP 在确认前已经提交 r13：

- 网页会提示预览过期；
- 即使网页尚未自动刷新，服务端也会以 409 拒绝旧事务；
- 用户必须重新生成预览。

## 验收步骤

1. 选择一条较长 connector，点击“新增偏移段”；
2. 确认管线增加偏移且端口绑定不变；
3. 点击“删除偏移段”，确认恢复原路径；
4. 制造多个共线折点，点击“清理共线折点”；
5. 选择一个阀门和相连管线，要求 Agent 向右移动并重新整理；
6. 确认只出现事务预览，文档 revision 不变化；
7. 检查 operation 清单后点击“确认应用”；
8. 确认历史来源为 `LLM`；
9. 再生成一个预览，在 MCP 修改文档后尝试确认，确认收到 revision 冲突。

## 验证范围

新增后端测试覆盖：

- validate 不写入文档；
- operation index 错误定位；
- dry-run 不增加 revision；
- 确认后历史来源为 `llm`；
- 过期预览返回 409。

路径算法完成了独立往返检查：新增偏移段后再删除，可恢复原始直线端点。

本切片未将“不同模型下自然语言局部移动、替换、删除和重连的稳定性”标记为完成；该项仍需真实模型矩阵和工程图纸回归验证。
